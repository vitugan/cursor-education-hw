Status line

Response Headers
alt-svc: — origin server can advertise
cache-control: — directives for caching mechanisms in both requests and responses (detail)
content-encoding: — used to compress the media-type, value indicates which encodings were applied to the entity-body
content-type: —indicate the media type of the recourse, it tells the client what the content type of the returned content actually is
content-security-policy: — allows web site administrators to control resources the user agent is allowed to load for a given page
date: — the date and time at which the message was originated
pragma: — is an implementation-specific header that may have various effects along the request-response chain
expires: — the date/time after which the response is considered stale
vary: — determines how to match future request headers to decide whether a cached response can be used rather than requesting a fresh one from the origin server
p3p: — Google's cookies
x-fb-debug: — Facebook cookies
server: — information about the software used by the origin server to handle the request
set-cookie: — used to send cookies from the server to the user agent
strict-transport-security: — response header (often abbreviated as HSTS) lets a web site tell browsers that it should only be accessed using HTTPS, instead of using HTTP
x-frame-options: — response header can be used to indicate whether or not a browser should be allowed to render a page in a <frame>, <iframe> or <object> .
x-xss-protection: — response header is a feature of Internet Explorer, Chrome and Safari that stops pages from loading when they detect reflected cross-site scripting (XSS) attacks
Request Headers
:authority: — pseudo-header field includes the authority portion of the target URI
:method: — pseudo-header, request method type
:path: — pseudo-header, server path for current page
schema: — pseudo-header indicates protocol (https/http)
accept-encoding: — advertises which content encoding, usually a compression algorithm, the client is able to understand
accept-language: — advertises which languages the client is able to understand, and which locale variant is preferred
cookie: — contains stored HTTP cookies previously sent by the server with the Set-Cookie header
